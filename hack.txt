hack技术：存在各种浏览器，在同类的浏览器中也存在着不同的版本，比如ie6 ie7等，不同的浏览器针对CSS的解析机制是不同的，因此会导致不同浏览器中页面的效果也不相同，
如何解决？可以针对某种浏览器进行样式设置，从而可以达到浏览器的显示效果一致，这种标识浏览器的方法就是hack
尽量使用html的方式去统一CSS样式，因为hack会导致维护成本增加。如果实在避免不了就是用选择器hack
*html *前缀只对IE6生效

*+html *+前缀只对IE7生效

@media screen\9{...}只对IE6/7生效

@media \0screen {body { background: red; }}只对IE8有效

@media \0screen\,screen\9{body { background: blue; }}只对IE6/7/8有效

@media screen\0 {body { background: green; }} 只对IE8/9/10有效

@media screen and (min-width:0\0) {body { background: gray; }} 只对IE9/10有效

@media screen and (-ms-high-contrast: active), (-ms-high-contrast: none) {body { background: orange; }} 只对IE10有效

[强制] 保证 favicon 可访问。
解释：
在未指定 favicon 时，大多数浏览器会请求 Web Server 根目录下的 favicon.ico 。为了保证 favicon 可访问，避免 404，必须遵循以下两种方法之一：
在 Web Server 根目录放置 favicon.ico 文件。
使用 link 指定 favicon。
示例：
<link rel="shortcut icon" href="path/to/favicon.ico">

favicon的好处：
1.一个简洁美观的favicon是网站博客的面子，在标签浏览器中他可以比title更加醒目，让读者很容易的识别出王爷，从而增加网站流量
2.在一定程度上减轻服务器的浏览器的流量带宽占用。因为一般为了提高网站的可用性，我们都会为自己的网站创建一个自定义404
错误，在这种情况下，如果网站没有相应的favicon.ico文件，每当有用户收藏网站/网页时，web服务器都会调用这个404文件，
并在网站的错误日志中记录，如果有的话这种情况是可以避免的。
